Auth in vanilla JavaScript

JWT with HTTP-only Refresh Token Cookie Demo

Login: Logs you in using the hardcoded user.
The hardcoded credentials are:
username: nicohorn
password: 12345

Access Protected Data: Makes a GET request to the /protected route which is only accessible if the client sends a request with the authorization header set, containing a valid JWT (access token) as the Bearer.
Refresh token: Makes a POST request to the /token endpoint. As the refresh token is an HTTP only cookie, the server will receive it along with the request body (in this case, nothing). This endpoint will run a function to refresh the access token, verifying first the validity of the refresh token.
Logout: Logs you out of the app by removing the refresh token and the access token.
